# Duplicate / correction flow

Session note:
- User logged a breakfast wrap + chorizo + fruit bowl.
- The same meal was then re-sent with a photo.
- Correct handling: treat as a likely duplicate/correction, not a fresh meal.
- Ask one of:
  - `correction` → replace the previous estimate/entry
  - `addition` → log again as another meal

Operational rule:
- When the latest user message matches a meal already logged today, prefer confirmation over auto-logging.
- A new photo of the same meal is not enough evidence to add a second entry.
- If the user says they did not eat a photographed side item (e.g. "remove the salad", "I didn't eat it"), revise the existing entry instead of appending a fresh meal.

Use this as a quick reminder when the same meal arrives twice in one day from a new angle or with a refined description.
